# 美食扭蛋机 - 设计文档

> 文档日期：2024-04-15
> 版本：v1.0

## 一、项目概述

### 1.1 项目背景
解决公司同事午餐「吃什么」的选择困难问题，通过趣味扭蛋机形式随机推荐食堂档口。

### 1.2 目标用户
- 一期：公司内部员工
- 二期：开放给所有互联网用户

### 1.3 核心功能
| 功能 | 说明 |
|------|------|
| 扭蛋机动画 | 保留原有动画，70%食堂/30%其他概率 |
| 食堂数据 | 现有档口 + 新增「3楼黄牛肉米线」 |
| 二维码 | 自动生成，支持下载分享 |
| 外卖入口 | Web弹窗提示 / 支付宝环境自动跳转 |
| 用户反馈 | 点赞/踩，匿名+设备限制防刷 |
| 数据统计 | 访问量、抽中分布、时段趋势、排行榜 |
| 管理后台 | 密码保护，数据可视化，CSV导出 |

## 三、数据库设计

### 3.1 spin_logs（抽奖记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键，自动生成 |
| food_name | TEXT | 抽中的档口名称 |
| food_emoji | TEXT | emoji表情 |
| floor | INTEGER | 楼层（0=其他美食） |
| device_id | TEXT | 设备指纹（匿名） |
| created_at | TIMESTAMP | 抽奖时间 |

### 3.2 feedbacks（用户反馈表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键，自动生成 |
| food_name | TEXT | 档口名称 |
| food_emoji | TEXT | emoji表情 |
| action | TEXT | 'like' 或 'dislike' |
| device_id | TEXT | 设备指纹 |
| created_at | TIMESTAMP | 反馈时间 |

**防刷逻辑**：同一 device_id 对同一档口每天只能反馈一次。

## 四、核心功能设计

### 4.1 扭蛋交互

1. 用户点击扭蛋按钮
2. 触发动画，随机滚动emoji
3. 根据70%/30%概率选择食堂/其他美食
4. 飞行扭蛋动画展示结果
5. 记录抽奖结果到数据库

### 4.2 外卖跳转逻辑

```javascript
if (isInAlipay()) {
    // 支付宝环境：直接跳转
    window.location.href = 'https://ur.alipay.com/_2m8rCylYokzSaPsBdYi9yv';
} else {
    // 其他环境：弹窗提示
    showToast('请打开淘宝闪购选购');
}
```

### 4.3 用户反馈

- 抽中结果卡片底部显示：👍 点赞 / 👎 没去
- 点击后记录，同一设备同一档口每天限一次
- 反馈后按钮变灰，提示「已反馈」
- 本地存储记录防止重复反馈

### 4.4 二维码生成

- 使用 qrcode.js 库
- 自动生成当前页面URL的二维码
- 支持点击复制链接分享

## 五、管理后台设计

### 5.1 访问方式
- 路径：`/admin.html`
- 入口：页面底部「管理入口」小字
- 密码：`admin123`（上线前修改）

### 5.2 功能模块

**1. 数据概览**
- 独立访客数
- 抽奖总次数
- 点赞数
- 好评率

**2. 趋势图**
- 今日：按小时分布
- 近7天：按日分布
- 近30天：按日分布
- 全部：按月分布

**3. 时段分布**
- 柱状图展示24小时抽奖分布
- 帮助了解高峰时段

**4. 档口排行榜**
- 按抽中次数排序
- 显示点赞/踩数量
- 计算好评率

**5. 最近记录**
- 最近20条抽奖记录
- 显示反馈状态

**6. CSV导出**
- 一键导出排行数据

## 十、风险与应对
